## ⚠️ Critical Rules — Empty File Bug Prevention

**This repo has a known recurring issue: agent edits can accidentally leave
`index.html` files at 0 bytes, producing a white/blank page in production.**

### Rule 1 — Never truncate or touch built HTML output files directly

Files inside `brand/` and `strategy/` are **generated build artefacts**.
Do NOT edit them directly unless you are mirroring a known-good source.

| Path | Source of truth |
|------|----------------|
| `brand/strategy/index.html` | `strategy-source/strategy/index.html` + Vite build |
| `brand/operations/index.html` | `strategy-source/operations/index.html` + Vite build |
| `brand/visual/index.html` | `strategy-source/visual/index.html` + Vite build |
| `strategy/index.html` | The older standalone brand-strategy Vite build |

### Rule 2 — If a build output HTML must be touched, mirror the shell exactly

The HTML shells in `brand/*/index.html` look like:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Extra …</title>
    <script type="module" crossorigin src="/brand/assets/<hash>.js"></script>
    <link rel="stylesheet" crossorigin href="/brand/assets/<hash>.css">
  </head>
  <body><div id="root"></div></body>
</html>
```

Never leave this file empty. If the build hasn't run yet, use the stub
template in `brand/operations/index.html` as a reference.

### Rule 3 — Audit for empty HTML before committing

Before any `git push`, run:

```bash
cd /path/to/extra-yash.github.io
find . -name "*.html" -not -path "./.git/*" -not -path "./strategy-source/node_modules/*" -exec wc -c {} \; | sort -n
```

Any file showing `0` bytes is a problem. Fix it before pushing.

### Rule 4 — To rebuild brand/* pages

```bash
cd strategy-source
npm run build          # outputs to ../brand/  (emptyOutDir: false)
```

Asset filenames are content-hashed. After a build, the `<script src>` and
`<link href>` hashes in the HTML shells will change — this is expected.

## Hand-authored files (never build-generated)

These files are edited directly and have **no build step**:

- `index.html` — landing page
- `style.css` — global stylesheet
- `script.js`, `dither.js`, `fluid.js`, `data.js` — vanilla JS modules
- `onboarding/index.html` — partner onboarding form

## Design System Quick Reference

- **Font — Body:** Space Mono
- **Font — Display:** Texturina (headings)
- **Accent colour:** `#e8f04a` (acid yellow-green)
- **Background:** `#0a0a0a`
- **Cards:** `rgba(255,255,255,0.03)` background, `backdrop-filter: blur(8px)`
- **Radius tokens:** `--radius: 16px`, `--radius-sm: 8px`
